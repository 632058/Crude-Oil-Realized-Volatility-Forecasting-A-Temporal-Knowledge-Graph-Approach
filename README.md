# Crude Oil Realized Volatility Forecasting with Temporal Knowledge Graph Embeddings

This repository contains a research workflow for crude oil realized volatility forecasting. The project combines traditional HAR-style volatility models with geopolitical information extracted from a temporal knowledge graph. The main interactive workflow is provided in `demo.ipynb`, which should be used for data processing, baseline training, GNN embedding analysis, rolling PCA experiments, evaluation, and result visualization.

## Project Overview

The goal of this project is to forecast crude oil realized volatility over multiple horizons by comparing standard volatility forecasting baselines against models enhanced with geopolitical graph embeddings.

The workflow includes:

- Preparing realized volatility, OVX, GPR, and GDELT-based geopolitical data
- Building HAR-family baseline models
- Training rolling Ridge regression forecasts
- Extracting daily GNN-based geopolitical context embeddings
- Applying rolling PCA to high-dimensional GNN embeddings
- Evaluating models with out-of-sample metrics
- Comparing GNN geopolitical signals with traditional GPR index signals

## Main Entry Point

Please use:

```text
demo.ipynb
```

as the main notebook for running the complete research workflow.

The notebook contains the primary end-to-end process, including:

1. Data preparation
2. Baseline model training
3. GNN embedding generation or loading
4. Rolling PCA feature construction
5. Forecast model evaluation
6. Result tables and plots

The standalone Python scripts are also included for reproducibility, but `demo.ipynb` is the recommended place to inspect, run, and modify the workflow step by step.

## Recommended Workflow in `demo.ipynb`

### 1. Install Dependencies

Install dependencies from `requirements.txt` before running the notebook. The project uses common scientific Python packages such as `pandas`, `numpy`, `scikit-learn`, `torch`, `duckdb`, `xgboost`, `matplotlib`, `seaborn`, `scipy`, and `statsmodels`.

### 2. Prepare Data

Use the data preparation section in `demo.ipynb` to build the combined realized volatility and geopolitical risk dataset.

The data preparation process:

- Loads realized volatility data from `data/rv.csv`
- Loads daily GPR data from `data/data_gpr_daily_recent.xls`
- Converts GPR columns into symbol-style rows
- Merges RV, OVX, and GPR records into a unified time-series CSV
- Saves the merged file as `data/rv_with_gpr.csv`
- Builds GNN-ready entity, relation, and time index mappings
- Writes processed graph records into the DuckDB database

The main downstream input for volatility forecasting is:

```text
data/rv_with_gpr.csv
```

### 3. Run Baseline Training

The baseline workflow trains HAR-family models over several forecast horizons.

Baseline models include:

- `HAR`
- `HAR-J`
- `HAR-OVX`
- `HAR-OVX-GPR`

The baseline script and notebook use rolling Ridge regression with standardized features. Forecast targets include:

- `Target_1D`
- `Target_1W`
- `Target_2W`
- `Target_3W`
- `Target_1M`
- `Target_2M`
- `Target_3M`

Baseline output files are written to:

```text
baseline_results/
```

Important generated files include:

```text
baseline_results/har_baseline_full_history_summary.csv
baseline_results/full_history_pred_HAR_Target_1D.csv
baseline_results/full_history_pred_HAR-OVX-GPR_Target_1M.csv
```

These outputs are used later for model comparison against the GNN-enhanced forecasting approach.

### 4. Extract GNN Geopolitical Embeddings

The GNN embedding workflow reads the prepared GDELT temporal knowledge graph from DuckDB and produces daily geopolitical context vectors.

The GNN model uses:

- Weighted graph convolution
- GRU-based temporal memory updates
- Optional Transformer encoder
- Optional LayerNorm
- Optional daily context fusion
- Poisson reconstruction loss on daily dyadic event counts

The default embedding extraction configuration tracks oil-related countries and entities, applies a CAMEO event-code filter, and outputs pooled daily context vectors.

Generated embeddings are saved under:

```text
gnn_embeddings/
```

Example output:

```text
gnn_embeddings/gnn_oil_context_full_mean_normT_fusT_from_20150401_cameo_ge10.csv
```

This CSV is then used by the rolling PCA and forecast comparison sections in `demo.ipynb`.

### 5. Run Rolling PCA Forecast Experiments

The notebook combines HAR-style volatility features with GNN embedding features. Because GNN embeddings may be high-dimensional, the workflow applies rolling PCA to avoid look-ahead bias.

The rolling PCA procedure:

- Uses only historical information available at each forecast date
- Fits PCA on the rolling training window
- Transforms the current test observation
- Adds PCA components as geopolitical embedding factors
- Trains a rolling Ridge regression model

The main GNN-enhanced model is reported as:

```text
HAR-OVX+RollPCA(k)
```

where `k` is the number of retained PCA components.

Generated prediction files are saved to:

```text
results/
```

Example files:

```text
results/full_history_pred_HAR-OVX+RollPCA(1)_Target_1D.csv
results/rolling_pca_sensitivity_results.csv
```

### 6. Evaluate Forecasting Performance

The notebook evaluates models using several metrics:

- `R2_Standard`
- `R2_OOS`
- `QLIKE`
- `MSE`
- test sample size
- evaluation start and end dates

The main comparison is between the GNN-enhanced model and the baseline HAR-family models, especially `HAR-OVX-GPR`.

The notebook prints summary tables and saves CSV files under `results/`.

### 7. Generate Plots and Comparisons

The final analysis sections in `demo.ipynb` generate visual comparisons, including:

- GNN PC1 contribution over time
- GPR contribution over time
- Normalized GNN versus GPR contribution comparison
- Forecast performance summaries

Generated figures are saved under:

```text
results/
```

## Standalone Scripts

Although `demo.ipynb` is the recommended workflow, the repository also includes standalone scripts.

### Data Preparation

```bash
python data_preparation.py
```

This prepares the graph mappings, GNN training table, and merged RV + GPR CSV.

### Baseline Training

```bash
python run_baseline.py
```

This trains HAR-family baselines and writes prediction and summary CSV files to `baseline_results/`.

### GNN Embedding Extraction

```bash
python get_embedding.py
```

This extracts daily GNN geopolitical context embeddings and writes them to `gnn_embeddings/`.

data/

## Notes

- Start from `demo.ipynb` for the full workflow.
- Run the data preparation cells before baseline training or GNN experiments.
- Make sure `data/rv_with_gpr.csv` exists before running forecasting models.
- Make sure the GNN DuckDB file and mapping file exist before extracting embeddings.
- Generated results may take time because the workflow uses rolling-window training and repeated model fitting across multiple forecast horizons.
